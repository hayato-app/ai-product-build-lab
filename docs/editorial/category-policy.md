# Category Policy

## Purpose

Categories help readers understand the primary theme of an article.

Each article should have one primary category. Tags should be used for secondary topics.

## Primary Categories

Use the following primary categories:

- AI開発入門
- AI開発ツール
- AI用語解説
- システム開発用語解説
- 生成AIアプリ開発
- Next.js AI開発
- OpenAI API
- AIエージェント開発
- エラー解決
- AIアプリ運用
- AIニュース解説
- AI開発トレンド

## Category Rules

- Use only one primary category per article.
- Use tags for secondary topics.
- Keep category names short and consistent.
- Do not create a new category unless at least 3 articles are expected.
- Avoid duplicate categories with similar meaning.
- Prefer a broader stable category over a narrow one-off category.

## Category Selection Guide

Use `AI開発入門` for beginner learning paths and first-step articles.

Use `AI開発ツール` for tool introductions and tool usage guides.

Use `AI用語解説` for AI-specific terminology explanations.

Use `システム開発用語解説` for general software and system development terminology.

Use `生成AIアプリ開発` for implementation articles about AI applications.

Use `Next.js AI開発` for articles where Next.js is the main implementation context.

Use `OpenAI API` for articles focused mainly on OpenAI API usage, errors, or design.

Use `AIエージェント開発` for agent design, tool execution, workflow, or safety topics.

Use `エラー解決` for troubleshooting articles where the main value is investigation and resolution.

Use `AIアプリ運用` for monitoring, cost control, logs, release, and improvement topics.

Use `AIニュース解説` for news commentary.

Use `AI開発トレンド` for trend analysis that is not tied to a single news event.

## Good Examples

Good category usage:

- Article: "OpenAI API timeout troubleshooting"
  - Category: `OpenAI API`
  - Tags: `エラー解決`, `AIアプリ運用`, `ログ`

- Article: "What is a vector database?"
  - Category: `AI用語解説`
  - Tags: `RAG`, `ベクトル検索`, `初心者`

- Article: "What is an environment variable?"
  - Category: `システム開発用語解説`
  - Tags: `環境変数`, `セキュリティ`, `初心者`

## Bad Examples

Avoid category usage like:

- Creating both `AI API Cost` and `AI APIコスト管理` for the same topic.
- Creating a category for a single tool name unless many articles are planned.
- Using `RAG開発` as a primary category when `AI用語解説`, `生成AIアプリ開発`, or `AIアプリ運用` is more appropriate.
- Using `AI APIコスト管理` as a primary category when `AIアプリ運用` or `OpenAI API` is enough.

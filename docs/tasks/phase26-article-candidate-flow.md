# Phase 26: Article Candidate Flow

## Purpose

Create the operating documents for selecting article candidates and turning
approved candidates into draft articles.

The approved direction is the combined A+B model:

- Manual approval based semi-automation.
- GitHub Issue driven operation.

## Scope

This phase documents the flow only.

It creates:

- `docs/operations/article-candidate-flow.md`
- `docs/article-candidates/README.md`

It may update:

- `AGENTS.md`
- `docs/editorial/article-operation-rules.md`
- `docs/operations/smartphone-codex-operation.md`

## In Scope

- Candidate list storage rules.
- Candidate format.
- Candidate evaluation criteria.
- Human approval points.
- Draft creation boundaries.
- Thumbnail default operation connection.
- Future automation boundaries.

## Out of Scope

This phase does not implement:

- Candidate generation scripts.
- Discord `/article-candidates` command.
- Automatic draft generation.
- Automatic publication.
- Search Console or Analytics integration.
- VPS deployment automation.
- AI API billing dependent generation.

## Completion Criteria

- The A+B flow is documented.
- Candidate list storage is documented.
- Candidate item format is documented.
- Human approval before draft creation is explicit.
- The relationship to draft review and thumbnail operation is clear.
- Future phases are identified.

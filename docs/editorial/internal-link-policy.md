# Internal Link Policy

## Purpose

Internal links help readers continue learning and help related articles support each other.

Internal links should be useful for the reader first. Do not force links only for SEO.

## Basic Rules

- Each article should include 2 to 4 related article links when possible.
- Each article should link to a relevant tool when possible.
- Links should be natural and contextually useful.
- Link to the next useful action for the reader.
- Avoid linking to unrelated articles only because they share a broad keyword.

## Preferred Internal Link Destinations

Prefer links to:

- Closely related articles
- Beginner explanations that support the current topic
- Troubleshooting articles that solve likely next problems
- Tool pages that help the reader calculate, inspect, or validate something
- Learning path or checklist articles

## Tool Link Rules

Important tool link:

```txt
/tools/ai-api-cost-estimator
```

Suggested anchor text:

- AI APIの概算コストを計算する
- AI API Cost Estimatorを使う
- 月間APIコストを見積もる

Use the tool link when the article discusses:

- AI API pricing
- token usage
- monthly cost estimation
- AI product operation
- release planning
- cost control

## Related Articles Section Format

Use this format when adding related articles:

```md
## 関連記事

- [記事タイトル](/articles/example-slug)
- [記事タイトル](/articles/example-slug)
```

## Links to Avoid

Avoid:

- Links that do not help the reader.
- Too many links in a short article.
- Repeated links to the same destination.
- Links that use vague anchor text such as "click here".
- Tool links that are not relevant to the article.

## この記事で扱うこと

AIアプリ開発を始めたい初心者向けに、最小構成で必要になる要素を整理します。最初から大きなSaaSを作るのではなく、画面、API、AI API、保存先、公開環境を小さく組み合わせる考え方を紹介します。

## 想定読者

- AIアプリを作ってみたいが、何から始めればよいか分からない人
- Next.jsやAPI Routesを使ってAI機能を試したい人
- 個人開発や小規模プロダクトの最初の構成を知りたい人

## 本文案

AIアプリの最小構成は、フロントエンド、サーバー側API、AI API、データ保存、公開環境の5つに分けると理解しやすくなります。

フロントエンドでは、ユーザーが入力するフォームと結果表示の画面を作ります。サーバー側APIでは、APIキーを隠したままAI APIを呼び出します。AI APIはテキスト生成、要約、分類、検索補助などの役割を持ちます。

最初の段階では、DBを必須にしない構成でも十分です。入力と回答を保存したい場合だけ、SQLite、PostgreSQL、Supabaseなどを検討します。公開環境はVercel、VPS、Cloudflareなどがありますが、初心者はまずデプロイしやすい環境を選ぶとよいです。

## 構成例

- 画面: Next.js App Router
- API: Route Handler
- AI API: OpenAI API または Gemini API
- 保存先: 最初はなし、必要になったらPostgreSQL
- 公開: Vercel または Docker on VPS

## まとめ

AIアプリは、最初から複雑な構成にする必要はありません。まずは「入力する」「AIに渡す」「結果を表示する」の3ステップを動かし、必要に応じて保存や認証を足していくのが現実的です。

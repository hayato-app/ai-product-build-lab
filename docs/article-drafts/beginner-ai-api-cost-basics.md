## 結論

AI APIのコストは、主に「1回で送る量」「1回で返す量」「何回実行されるか」で決まります。

最初から正確な金額を当てる必要はありません。公開前に見るべきなのは、次の3つです。

| 見ること | 理由 |
| --- | --- |
| 1回あたりの入力と出力 | 1回のAPI呼び出し単価に影響する |
| 1ユーザーあたりの実行回数 | 利用者が増えたときの費用に影響する |
| 月間利用量 | 公開後の上限や予算判断に必要 |

![AI APIコストを入力、出力、実行回数、月間利用量に分けて見積もる図](/images/drafts/beginner-ai-api-cost-basics-thumbnail.svg)

## 対象読者

- AI APIの料金がなぜ増えるのか分からない人
- AIアプリを公開する前に月額費用をざっくり見積もりたい人
- トークン、実行回数、モデル単価の関係を理解したい人
- APIコストが急に増えない設計を考えたい人

## 前提

この記事では、特定サービスの料金表そのものは扱いません。

料金表は変わるため、ここでは見積もり方を中心に説明します。金額を決める直前には、必ず利用するAI APIの公式料金ページを確認してください。

## AI APIコストを分けて考える

AI APIの費用は、細かく見るといくつかの要素に分かれます。

| 要素 | 意味 | コストへの影響 |
| --- | --- | --- |
| 入力トークン | AIに送る文章、指示、履歴、検索結果など | 長いほど増える |
| 出力トークン | AIが返す文章 | 長いほど増える |
| 実行回数 | APIを呼び出す回数 | 多いほど増える |
| モデル単価 | 利用するモデルごとの価格 | 高性能モデルほど高くなりやすい |
| リトライ | 失敗時の再実行 | 失敗が多いほど増える |

トークンは、AIが文章を処理するための単位です。日本語の文字数と完全に一致するわけではありませんが、文章が長いほどトークンも増える、と理解すれば十分です。

## ざっくり見積もる式

最初は、次のように分けると見積もりやすくなります。

```txt
1回あたりのコスト
= 入力トークンのコスト + 出力トークンのコスト

1日あたりのコスト
= 1回あたりのコスト × 1日の実行回数

1か月あたりのコスト
= 1日あたりのコスト × 30
```

正確な計算よりも、どの変数が大きいかを見ることが重要です。

## 見積もりに必要な項目

公開前には、最低でも次の項目を書き出します。

| 項目 | 例 |
| --- | --- |
| 1回の入力量 | 短い質問、長い文章、ファイル内容など |
| 1回の出力量 | 1段落、要約、レポート、JSONなど |
| 1人あたりの実行回数 | 1日1回、1日10回など |
| 想定ユーザー数 | 10人、100人、1000人など |
| 失敗時のリトライ | 最大1回、最大3回など |
| 利用モデル | 用途に合うモデルを選ぶ |

これらを決めずに公開すると、利用が増えたときに費用の説明が難しくなります。

## コストが増えやすいパターン

| パターン | なぜ増えるか | 対策 |
| --- | --- | --- |
| 会話履歴を全部送る | 入力トークンが増える | 必要な履歴だけ送る |
| 返答を長くしすぎる | 出力トークンが増える | 出力形式や文字数を指定する |
| ボタン連打を許す | 実行回数が増える | 実行中はボタンを無効化する |
| 無制限リトライ | 失敗時に費用が増える | 最大回数と待機時間を決める |
| 全部高性能モデルに投げる | モデル単価が高くなる | 用途別にモデルを分ける |

## ツールで概算する

手計算が難しい場合は、[AI API Cost Estimator](/tools/ai-api-cost-estimator) のようなツールで概算します。

見積もりツールを使う場合も、入力文字数、出力文字数、実行回数、月間利用量の前提は自分で決める必要があります。

## 公開前チェックリスト

- [ ] 入力文字数の最大長を決めている
- [ ] 出力の長さや形式を指定している

## まとめ

AI APIコストは、ユーザー数だけで決まるわけではありません。

1回あたりの入力、出力、実行回数、リトライ、モデル選択で大きく変わります。公開前には、細かい金額を一度で当てるよりも、費用が増える場所を分解しておくことが重要です。

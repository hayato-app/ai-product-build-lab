# Content Improvement Opportunities

This file is generated by:

```bash
node scripts/analyze-content-opportunities.mjs data/analytics/search-console.csv
```

Raw analytics CSV files should be stored under:

```txt
data/analytics/
```

The `data` directory is ignored by Git. Do not commit raw Search Console or
Analytics exports.

## Workflow

1. Export Search Console or Analytics data as CSV.
2. Save the CSV under `data/analytics/`.
3. Run `scripts/analyze-content-opportunities.mjs`.
4. Review the generated opportunities.
5. Create focused `article-review` Issues for selected articles.

## Expected CSV Columns

Minimum:

- `page`
- `clicks`
- `impressions`
- `ctr`
- `position`

Optional:

- `engagedSessions`
- `averageEngagementTime`
- `toolClicks`

## Opportunity Types

The script looks for:

- High impressions with low CTR.
- Search position between 11 and 30.
- Traffic with short engagement.
- Articles with no recorded tool clicks.
- Articles with few internal links.

Run the script with real exported data to replace this template with a generated
report.
